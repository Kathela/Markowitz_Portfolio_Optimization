# Markowitz Portfolio Optimization

A production-ready Python implementation of **Modern Portfolio Theory (MPT)** following Harry Markowitz's seminal 1952 paper *"Portfolio Selection"* — built as a portfolio project for the Data Analyst → Quant Analyst transition.

---

## Overview

This project answers a fundamental question in finance: **given a set of risky assets, how should an investor allocate capital to maximise return for a given level of risk?**

Markowitz showed that the answer is not to simply pick the highest-returning assets, but to exploit the *correlation structure* between assets. By combining assets that do not move perfectly in lockstep, a portfolio can achieve the same expected return at lower risk — the "free lunch" of diversification.

## Methodology

### 1. Data
- **Source**: Yahoo Finance via `yfinance` — split- and dividend-adjusted (total-return) prices
- **Period**: 3 years of daily adjusted-close prices
- **Missing data**: Forward-filled then dropped, keeping the longest common history

### 2. Expected Returns & Covariance
```
μ_annual = μ_daily × 252      (linearity of expectation)
Σ_annual = Σ_daily × 252      (variance scales linearly with time)
```
Using **arithmetic** (not log) returns because portfolio return is *exactly* a weighted sum of asset arithmetic returns.

### 3. Monte Carlo Simulation
- 10,000 random long-only weight vectors drawn from the uniform Dirichlet distribution
- Each portfolio's return, volatility, and Sharpe ratio is computed
- Provides a visual approximation of the full feasible set

### 4. Portfolio Optimisation (scipy SLSQP)
Two optimal portfolios are found via constrained quadratic programming:

| Portfolio | Objective | Character |
|-----------|-----------|-----------|
| **Max Sharpe** | Maximise `(R_p - R_f) / σ_p` | Tangency portfolio; highest risk-adjusted return |
| **Min Variance** | Minimise `√(wᵀΣw)` | GMV portfolio; most conservative allocation |

Constraints: weights sum to 1, all weights ∈ [0, 1] (long-only, no leverage).

The optimiser automatically retries with tighter bounds `[0.01, 0.50]` if the first pass does not converge.

### 5. Efficient Frontier
Traces 200 minimum-variance portfolios for target returns from the GMV return to the maximum single-asset return. Each point solves a constrained optimisation problem.

### 6. Risk-Free Rate
`R_f = 5%` (approximate annualised US 10-year Treasury yield, 2024).

## Financial Concepts

**Modern Portfolio Theory (Markowitz, 1952)**
The foundational insight is that portfolio variance depends not just on individual asset volatilities, but on how assets *co-vary*. By selecting a portfolio on the efficient frontier, an investor obtains the maximum expected return for their chosen level of risk — or equivalently, the minimum risk for their target return.

**The Efficient Frontier**
Every portfolio that cannot be dominated — where no alternative offers higher return for the same risk or lower risk for the same return. Portfolios below the frontier are wasteful (sub-optimal).

**Sharpe Ratio**
`S = (R_p - R_f) / σ_p`
Measures excess return earned per unit of total risk. The higher the Sharpe ratio, the more return the investor receives for bearing each unit of risk.

**Diversification**
When assets have correlations less than 1, combining them reduces portfolio variance more than proportionally to weight. The correlation heatmap shows which asset pairs offer the greatest diversification benefit.

## Assumptions & Limitations

- **Historical returns predict future returns** — this is an assumption of the model; actual forward returns will differ
- **Normal distribution of returns** — heavy tails and fat-tail events are not modelled
- **Long-only, no leverage** — short-selling and leverage are excluded
- **No transaction costs or taxes** — practical implementation costs are ignored
- **Static covariance** — the covariance matrix is assumed constant over the holding period
